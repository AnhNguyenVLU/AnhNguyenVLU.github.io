1. What is CSS?

CSS (Cascading Style Sheets) is a styling language used to describe the presentation of HTML documents, controlling aspects like colors, fonts, layouts, and spacing. It separates content (HTML) from design (CSS), enabling easier maintenance and consistent styling across multiple pages.

Purpose: CSS enhances the visual appeal, ensures cross-device compatibility, and improves user experience by making websites visually engaging and accessible.

How it works: CSS uses rules comprising selectors (to target HTML elements) and declarations (property-value pairs). Rules cascade, meaning later rules can override earlier ones based on specificity and order.

Real-World Use Case: CSS is used to style e-commerce websites, ensuring product cards have consistent fonts, colors, and spacing, enhancing brand identity.

Example 1: Styling a heading and paragraph

h1 {
  color: #34C759;
  font-size: 2.5em;
  text-align: center;
}
p {
  font-size: 1.1em;
  line-height: 1.8;
}

Tip: Use CSS preprocessors like Sass to manage large stylesheets with variables and mixins, improving scalability for complex projects.

Note: Always link external CSS files for large projects to keep HTML clean and enable caching for faster page loads.

2. CSS Syntax

CSS syntax consists of a selector and a declaration block. The selector targets HTML elements, and the declaration block, enclosed in curly braces {}, contains property-value pairs separated by colons : and terminated by semicolons ;.

Structure: selector { property: value; property: value; }

Common Mistakes: Missing semicolons or curly braces can cause styles to fail. Always validate syntax using tools like CSSLint.

Example 1: Styling a paragraph with multiple properties

p {
  color: #333;
  font-size: 16px;
  text-align: justify;
  padding: 10px;
}

This paragraph demonstrates CSS syntax with multiple properties for text styling and spacing.

Practical Application: Use consistent syntax in team projects by adopting style guides (e.g., Airbnb CSS Guide) to ensure readability and maintainability.

Example 2: Combining multiple selectors

h1, h2, .highlight {
  color: #34C759;
  font-weight: bold;

3. Basic CSS Selectors

Selectors identify HTML elements to style. They range from simple (tag names) to complex (combinators and pseudo-classes).

Types of Selectors:

  • Type Selector: Targets elements by tag name.

    • Example 1: Styling all h1 elements

    h1 {
  color: navy;
  text-transform: uppercase;
}

    Type Selector

  • Class Selector: Targets elements with a specific class.

    • Example 2: Styling elements with class .intro

    .intro {
  background-color: #e7f3e7;
  padding: 15px;
}

    This is an introductory text.

  • ID Selector: Targets a single element with a unique ID.

    • Example 3: Styling an element with ID main-header

    #main-header {
  font-size: 28px;
  border-bottom: 2px solid #34C759;
}

    Main Header

  • Universal Selector: Targets all elements.

    • Example 4: Resetting margins and padding

    * {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

    Text with no margin or padding.

  • Attribute Selector: Targets elements by attribute values.

    • Example 5: Styling inputs with type text

    input[type="text"] {
  border: 1px solid #34C759;
  padding: 5px;
}

Example 6: Using a pseudo-class for interactivity

button:hover {
  background-color: #2a9d4a;
  transform: scale(1.05);
}

Tip: Combine selectors (e.g., .class1.class2) for specificity, but avoid overly complex selectors to maintain readability.

4. CSS How To

CSS can be applied in three ways, each suited for different use cases. Understanding their precedence and best practices is key to effective styling.

Three Methods:

  • External CSS: Uses a separate .css file linked via <link>. Ideal for site-wide styles.

    • Example 1: Linking and styling with an external file

    <link rel="stylesheet" href="styles.css">

    Content of styles.css:

    body {
  background-color: #f8f9fa;
}
h1 {
  color: #34C759;
  margin: 20px;
}

    External CSS Heading

    Styled via external CSS.

  • Internal CSS: Uses a <style> tag in the <head>. Suitable for single-page styles.

    • Example 2: Internal styles for a landing page

    <style>
  body {
    background-color: #e7f3e7;
  }
  h1 {
    color: #2a9d4a;
    text-align: center;
  }
</style>

    Internal CSS Heading

    Styled via internal CSS.

  • Inline CSS: Uses the style attribute on elements. Best for quick, one-off styles.

    • Example 3: Inline styling for a unique element

    <h1 style="color: #34C759; font-size: 24px;">Inline Heading</h1>
<p style="color: #333; font-style: italic;">Inline Paragraph</p>

    Inline Heading

    Inline Paragraph

Cascading Order: Styles are applied in this priority (highest to lowest):

  1. Inline styles
  2. Internal/External styles (based on specificity and order)
  3. Browser defaults

Tip: Minimize inline CSS to avoid maintenance issues. Use CSS classes for reusable styles.

5. CSS Comments

CSS comments, written between /* */, are ignored by browsers. They document code, explain decisions, or temporarily disable styles.

Benefits: Improve team collaboration, aid debugging, and maintain code clarity in large projects.

Example 1: Using comments for organization

/* Navigation Bar Styles */
nav {
  background-color: #34C759;
  /* padding: 15px; */ /* Disabled for now */
  color: white;
}

/* Reset default margins */
* {
  margin: 0;
}
Sample Navigation

Text with reset margin.

Example 2: Commenting for debugging

/* Testing button styles */
button {
  background-color: #34C759;
  /* background-color: red; */ /* Alternative color for testing */
  padding: 10px;
}

Tip: Use multi-line comments for section headers and single-line comments for inline notes. Avoid over-commenting obvious styles.

6. CSS Colors

CSS supports multiple color formats: names, hexadecimal, RGB, RGBA (with opacity), HSL, and HSLA. Choosing the right format depends on design needs and accessibility.

Color Formats:

  • Name: Simple, but limited (e.g., red).
  • Hex: Precise, widely used (e.g., #ff0000).
  • RGB: Defines red, green, blue values (e.g., rgb(255, 0, 0)).
  • RGBA: Adds alpha for opacity (e.g., rgba(255, 0, 0, 0.5)).
  • HSL: Hue, saturation, lightness for intuitive adjustments.

Example 1: Applying different color formats

p.name {
  color: red;
}
p.hex {
  background-color: #ff0000;
  color: white;
}
p.rgb {
  color: rgb(255, 0, 0);
}
p.rgba {
  background-color: rgba(255, 0, 0, 0.3);
}
p.hsl {
  color: hsl(0, 100%, 50%);
}

Properties:

  • background-size: Scales image (cover, contain).

9. CSS Margins

Margins create space outside an element’s border, controlling layout spacing. Shorthand syntax: margin: top right bottom left;.

Margin Collapsing: Vertical margins between adjacent elements may combine, taking the larger value.

Example 1: Applying margins for spacing

p {
  margin: 15px 25px;
  border: 1px solid #34C759;
}

Text with Margins

Example 2: Centering with auto margins

Real-World Use Case: Margins are used in card layouts to space out content blocks in blogs or dashboards.

Tip: Use negative margins cautiously to overlap elements, but prefer transforms for positioning.

10. CSS Padding

Padding adds space inside an element, between content and border. Unlike margins, padding does not collapse.

Shorthand: padding: top right bottom left;

Example 1: Adding padding for content spacing

  padding: 20px 30px;
  border: 1px solid #34C759;

11. CSS Height/Width

Control element dimensions with height, width, min-height, max-width, etc., using units like px, %, vw, or rem.

Units:

  • px: Fixed pixels.
  • %: Relative to parent.
  • vw/vh: Viewport width/height.
  • rem/em: Relative to root/element font size.

Example 1: Setting responsive dimensions

div {
  width: 60%;
  height: 150px;
  max-width: 500px;
  min-height: 100px;
  background-color: #e7f3e7;
}
Responsive Box

Example 2: Using viewport units

section {
  height: 50vh;
  width: 80vw;
  background-color: #f8f9fa;
}
Viewport-Sized Section

Real-World Use Case: Width and height are used in responsive image galleries to ensure consistent thumbnail sizes.

Tip: Use min-height for containers to prevent content clipping on dynamic pages.

12. The CSS Box Model

The box model defines an element’s structure: content, padding, border, and margin. box-sizing: border-box; includes padding and border in the specified width/height.

Components:

  • Content: The inner content (text, images).
  • Padding: Space inside the border.
  • Border: Surrounds padding.
  • Margin: Space outside the border.

Example 1: Applying the box model

* {
  box-sizing: border-box;
}
div {
  width: 300px;
  padding: 15px;
  border: 5px solid #34C759;
  margin: 20px;
  background-color: #e7f3e7;
}

Real-World Use Case: The box model is critical in form design, ensuring inputs have consistent padding and borders for usability.

Tip: Apply box-sizing: border-box; globally to simplify sizing calculations.

13. CSS Outline

Outlines are drawn outside borders, don’t affect layout, and are useful for focus states or debugging. Properties include outline-style, outline-width, and outline-color.

Example 1: Adding an outline for focus

button {
  outline: 2px dashed #34C759;
  outline-offset: 3px;
  padding: 10px;
}

Example 2: Custom focus outline for accessibility

input:focus {
  outline: 3px solid #34C759;
  outline-offset: 2px;
}

Tab to the input to see the focus outline.

Accessibility Tip: Ensure outlines are visible for keyboard users to maintain navigability. Use at least 2px thickness for visibility.

Tip: Use outlines instead of borders for temporary debugging to avoid layout shifts.

Specificity Hierarchy:

  • Inline styles
  • ID selectors
  • Class, attribute, pseudo-class selectors
  • Type selectors

29. CSS Flexbox

Flexbox is a one-dimensional layout system for aligning items in rows or columns, ideal for navigation bars, forms, or card layouts.

Key Properties:

  • flex-direction: Row, column, row-reverse.
  • flex-wrap: Wrap, nowrap.
  • justify-content: Space-between, center, flex-end.
  • align-items: Stretch, center, baseline.

Example 1: Responsive flex layout

.flex-container {
  display: flex;
  flex-wrap: wrap;
  justify-content: space-around;
  gap: 15px;
}
.card {
  flex: 1 1 200px;
  background-color: #e7f3e7;
  padding: 15px;

30. CSS Grid

CSS Grid is a two-dimensional layout system for creating complex layouts with rows and columns.

Key Properties:

  • grid-template-columns: Defines column sizes.
  • grid-template-rows: Defines row sizes.
  • grid-gap: Sets spacing between items.
  • grid-template-areas: Names areas for placement.

Example 1: Photo gallery grid

.grid-container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(150px, 1fr));
  gap: 10px;
}
.grid-item {
  background-color: #e7f3e7;
  padding: 10px;
}
